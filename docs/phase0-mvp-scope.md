# Phase 0 — MVP Scope and Product Framing

## Goal
Turn the presales idea into a buildable MVP by selecting one primary persona, one core user journey, and a lean feature set.

## 1. Primary persona
- Who is the first user of this product?
- What job are they trying to complete?
- What pain point is the product solving?

## 2. Core user journey
Describe the first end-to-end flow to build.

Example:
1. User signs in
2. User creates or loads a key item
3. User performs the main action
4. User sees the result or outcome

## 3. MVP feature set
### Must-have
- Feature A
- Feature B
- Feature C

### Nice-to-have later
- Feature D
- Feature E

## 4. Out of scope for v1
- Large-scale integrations
- Advanced automation
- Complex admin workflows
- Multi-role enterprise features

## 5. Success metrics
- Activation: how many users complete the first key action
- Retention: how often they return
- Core completion rate: how many users finish the main flow

## 6. Open questions
- What is the exact primary problem to solve?
- Which persona should be prioritized first?
- What is the most important action users need to complete?
- Should AI features be part of v1 or v2?

## Recommended next step
Choose one persona and one core workflow, then turn that into a single sprint backlog.
